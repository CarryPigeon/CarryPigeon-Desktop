# 用户信息页设计

版本 v0.2

日期 2026-4-3


# 1. 概述

本文档定义了 CarryPigeon 桌面客户端中**用户信息悬浮卡**（User Profile Popover）的设计规范和实现要求。

该功能用于在聊天界面快速查看其他用户的基本信息，提升用户体验。


# 2. 功能需求

## 2.1 触发方式

悬浮卡支持两种触发方式：

1. **左键点击触发**：用户在聊天界面中左键点击任意用户头像时，弹出用户信息悬浮卡
2. **悬停触发**：用户将鼠标指针停留在用户头像上方超过 `300ms` 后，自动弹出悬浮卡

## 2.2 关闭方式

满足以下任一条件时自动关闭悬浮卡：

1. **点击外部**：用户点击悬浮卡区域以外的任意位置时关闭
2. **鼠标离开**：鼠标离开悬浮卡区域和触发头像区域超过 `500ms` 后自动关闭
3. **滚动页面**：聊天区域滚动时自动关闭（可选优化）

## 2.3 功能特性

- 支持查看**自己**和**其他用户**的信息
- 支持自定义背景图片（仅对自己可编辑）
- 支持点击跳转至完整的用户信息页面
- 支持复制用户 ID 和邮箱到剪贴板
- 响应式布局，适应不同位置弹出


# 3. 数据结构

## 3.1 Props（组件输入）

```ts
interface UserProfilePopoverProps {
  // 用户 ID，必填
  userId: string;
  // 用户显示名称，可选（若不提供则从后端获取）
  username?: string;
  // 用户邮箱，可选（若不提供则从后端获取）
  email?: string;
  // 用户简介/签名，可选（若不提供则从后端获取）
  bio?: string;
  // 用户头像 URL，可选
  avatarUrl?: string;
  // 背景图片 URL，可选
  backgroundUrl?: string;
  // 触发方式：'click' | 'hover'，默认 'click'
  trigger?: 'click' | 'hover';
  // 弹出位置：'top' | 'bottom' | 'left' | 'right'，默认根据空间自动选择
  placement?: Placement;
}
```

## 3.2 后端数据结构

从用户服务获取的数据格式：

```ts
interface UserProfile {
  id: string;
  username: string;
  email: string;
  bio: string;
  avatarUrl: string | null;
  backgroundUrl: string | null;
  isCurrentUser: boolean;
}
```


# 4. 布局设计

## 4.1 整体布局

```
┌─────────────────────────────────────┐
│  [背景图片区域]                       │
│                                     │
│                   ┌─────────────┐    │
│                   │  圆形头像   │    ← 头像上半圆遮挡背景
│                   ◔─────────────┘    │
└─────────────────────────────────────┘
        ↓ 留出间距
┌─────────────────────────────────────┐
│  昵称                               │
│  邮箱 [复制按钮]                     │
│                                     │
│  简介/签名                           │
│                                     │
│  [查看完整资料]  [编辑资料]（仅自己） │
└─────────────────────────────────────┘
```

## 4.2 尺寸规范

| 区域 | 尺寸 | 说明 |
|------|------|------|
| 悬浮卡整体 | 宽度 `320px`，最大高度 `400px` | 固定宽度，高度自适应内容 |
| 背景区域 | 高度 `120px` | 宽度等于卡片宽度 |
| 用户头像 | 直径 `80px` | 圆形 |
| 头像位置 | 圆心在背景底边向上 `40px`，左边距 `20px` | 上半圆遮挡背景区域 |
| 信息区域 | 上边距 `60px`，内边距 `16px 20px 20px` | 背景为主题色面板色 |

## 4.3 视觉层次

1. **背景图片区域**
   - 占满卡片顶部宽度
   - 圆角为 `12px 12px 0 0`（跟随卡片整体圆角）
   - 未设置背景时，使用渐变默认背景 `linear-gradient(135deg, #667eea 0%, #764ba2 100%)`
   - 背景图片显示模式：`object-fit: cover`

2. **用户头像**
   - 圆形边框 `4px` 实心，颜色与卡片背景相同 (`--cp-panel`)
   - 阴影增强层次感
   - 点击头像可跳转至完整用户信息页

3. **信息区域**
   - 背景：`--cp-panel` 变量
   - 整体圆角：`12px`
   - 阴影：`--cp-shadow`

4. **文字样式**

| 元素 | 字体 | 大小 | 字重 | 颜色 |
|------|------|------|------|------|
| 昵称 | `--cp-font-display` | `20px` | `900` | `--cp-text` |
| 邮箱 | 默认 | `14px` | `400` | `--cp-text-muted` |
| 简介 | 默认 | `14px` | `400` | `--cp-text` |
| 按钮文字 | 默认 | `12px` | `600` | `--cp-primary` |


# 5. 交互设计

## 5.1 编辑背景图片（仅当前用户可编辑）

- 背景图片区域右上角有一个**相机图标按钮**
- 点击按钮弹出文件选择器，支持 `png`, `jpg`, `jpeg`, `webp` 格式
- 文件大小限制：**最大 5MB**
- 选择图片后立即上传并预览
- 上传成功后保存到用户设置

## 5.2 邮箱和 UID 复制

- 邮箱显示旁边有复制图标按钮
- 点击后复制到剪贴板，显示"已复制"提示
- 使用现有的 `MonoTag` 组件实现（已有 copyable 功能）

## 5.3 跳转操作

- 点击"查看完整资料"按钮 → 路由跳转至 `/user-info?uid={userId}`
- 点击"编辑资料"按钮 → 路由跳转至 `/settings/profile`
- 点击用户头像 → 同样触发"查看完整资料"跳转

## 5.4 动画效果

- 弹出：透明度从 0 到 1 + 微小缩放 (`scale(0.95)` → `scale(1)`)，时长 `200ms`
- 关闭：反向动画，时长 `150ms`
- 使用 CSS `transition` 实现


# 6. 组件架构

## 6.1 文件位置

按照项目功能边界原则，放置在：

```
src/features/account/profile/
├── presentation/
│   └── components/
│       └── UserProfilePopover.vue   # 主组件
└── api/
    └── getUserProfile.ts             # 获取用户信息 API
```

## 6.2 数据流

1. 如果 props 已经提供了完整信息，直接使用 props（用于快速显示）
2. 如果 props 不完整，仅提供了 userId → 调用 API 从后端获取完整信息
3. 加载时显示骨架屏占位

## 6.3 弹出层实现

使用 `@popperjs/core`（项目已通过 TDesign 依赖）实现定位：

- 自动计算弹出位置，避免超出视口
- 支持箭头指向触发元素
- 使用原生 Vue Teleport 将弹出层渲染到 body 下避免层级问题


# 7. 权限与状态

| 场景 | 可编辑背景 | 显示编辑按钮 | 说明 |
|------|------------|--------------|------|
| 查看自己 | ✅ 是 | ✅ 显示 | 当前用户可以编辑自己的背景 |
| 查看他人 | ❌ 否 | ❌ 隐藏 | 只能查看，不能编辑 |

## 7.1 加载状态

- API 请求中显示骨架屏占位
- 骨架屏对应位置用 pulsing 动画

## 7.2 错误处理

- API 获取失败时显示"加载失败"文本
- 保留重试按钮


# 8. 样式规范

- 使用 SCSS 预处理器
- 使用全局 CSS 变量 (`--cp-*`) 适配主题
- 所有样式使用 `scoped`
- 圆角统一使用 `12px`（卡片整体），按钮使用 `999px`（药丸形状）
- 间距遵循项目 4px 网格原则（4px, 8px, 12px, 16px, 20px, ...）


# 9. 可访问性

- 弹出层点击 `Esc` 键可关闭
- 按钮添加适当的 `aria-label`
- 颜色对比度满足 WCAG AA 标准


# 10. 集成点

## 10.1 在消息气泡中集成

每个消息气泡的头像外层包裹 `UserProfilePopover` 组件，配置：
- `trigger="hover"`（悬停触发）
- `placement="bottom-start"` 或自动

## 10.2 在成员列表中集成

成员列表项的头像外层包裹 `UserProfilePopover` 组件，配置：
- `trigger="click"`（点击触发）
- 根据位置自动选择 placement


# 11. 技术依赖

- Vue 3 Composition API (`<script setup lang="ts">`)
- `@popperjs/core` - 弹出定位
- TDesign Vue Next - 图标、按钮组件
- 项目现有 `MonoTag` 组件 - 复制功能
- Vue Router - 路由跳转
