# 15｜键盘与无障碍（桌面效率 + A11y）

版本：v1.0  
日期：2026-01-31

## 1. 全局快捷键（建议）

- `Ctrl/Cmd+K`：Quick Switcher（服务器/频道/插件搜索与跳转）
- `Esc`：关闭所有浮层（菜单/抽屉/Modal/Popover），并可选“回到输入框”
- `Ctrl/Cmd+P`：打开插件中心（与 Patchbay 隐喻强相关）
- `Ctrl/Cmd+,`：打开设置

聊天输入：
- `Enter`：发送
- `Shift+Enter`：换行

## 2. 焦点顺序（建议）

主界面（/chat）：
1) Server Rail（RackItem 列表）
2) Channel Rail（搜索 → 频道列表 → 当前用户区）
3) Top Console（搜索/连接状态）
4) Signal Pane（消息列表，可选跳转）
5) Composer Host（domain selector → 输入 → 发送）
6) Members Rail

原则：
- 不要让“纯装饰元素”进入 Tab 序列
- ContextMenu/Drawer 打开后必须 focus trap（并支持 Esc 关闭）

## 3. ARIA 与可读性

菜单：
- `role="menu"`，菜单项 `role="menuitem"`（现有组件可对齐）
- 右键菜单必须支持键盘触发（例如在气泡上 `Shift+F10`）

Badge：
- required/failed 不仅靠颜色：加 `aria-label` 或文本

## 4. 动效与可访问性

- 所有呼吸/闪烁动效都必须在 `prefers-reduced-motion: reduce` 下关闭。
- 红色闪烁频率避免过快（避免光敏不适）。
