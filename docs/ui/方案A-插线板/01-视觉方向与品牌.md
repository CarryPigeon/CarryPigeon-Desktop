# 01｜视觉方向与品牌（Modular Patchbay）

版本：v1.0  
日期：2026-01-31

## 1. 核心隐喻（让抽象变直觉）

### 1.1 服务器 = 机柜（Rack）

- 每个服务器是一列“机柜条”，包含：名称/Socket/连接灯（LED）/插件供电状态。
- 切换服务器像切换机柜：**插件集合/版本/缓存**的隔离变得“可感知”。

### 1.2 插件 = 模块（Module）

- 插件中心以“模块机柜”的方式呈现：每个插件卡片是一个模块面板（带端口与信息铭牌）。
- 安装/启用/失败/更新=模块插入/上电/保险丝熔断/热插拔（对应 PRD 的状态机与回滚）。

### 1.3 Domain = 信号线路（Cable）

- 消息的 `domain` 用“线缆颜色”表达（例如 Core:Text = 青绿；Math:Formula = 蓝；Poetry = 紫红）。
- 未知 domain = “未接线信号”：展示 preview（若有）+ “安装模块以解码”按钮。

### 1.4 required gate = 供电锁扣（Power Latch）

- required 缺失不是“错误”，而是“供电锁扣未闭合”，阻止登录但提供明确修复路径（PRD 7.1）。

## 2. 视觉基调

- 背景：石墨黑 + 细网格（像机柜刻度），低对比；内容面板用半透明深色玻璃。
- 强调：高饱和“线缆色”只用于 domain、进度、连接状态，避免整屏花。
- 形状：模块面板为硬朗圆角矩形；按钮与 Badge 更像工业标签（Label）。

## 3. 可访问性与可用性

- 对比度：正文/关键按钮确保 WCAG AA（暗色主题尤其要守住）。
- 色彩不作为唯一信息：domain 颜色必须配合文字标签/图标；required/failed 需要形状与文案同时表达。
- 键盘效率（桌面客户端价值）：全局 `Ctrl/Cmd+K` 搜索；`Esc` 关闭所有浮层；输入区 `Enter` 发送、`Shift+Enter` 换行（与现有一致）。

## 4. 与现有工程的兼容策略

- 仍使用 CSS 变量作为 Token 单一真相；可通过 `:root[data-theme="patchbay"]` 切换主题，避免一次性替换全部样式。
- 仍兼容 tdesign-vue-next：优先覆盖 `--td-*` 变量，组件层尽量少写“局部主题”。
