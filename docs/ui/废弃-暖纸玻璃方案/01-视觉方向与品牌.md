# 01｜视觉方向与品牌

版本：v1.1  
日期：2026-01-31  
适用：Desktop（主窗口 + 子窗口/Popover）

## 1. 设计立场（与产品一致）

CarryPigeon 的核心记忆点不是“又一个聊天软件”，而是：

- **自建服务器**：强烈的“归属服务器”的存在感（服务器隔离、不同插件集合/版本）。  
- **插件生态**：消息可以来自不同 domain，UI 必须为“未知/未装插件”留出优雅降级。  
- **安全边界**：required 插件是 gate，体验上要“阻止但不挫败”，给用户可达的修复路径。

## 2. 视觉概念：Postal Atelier（邮局工坊）

方向：**暖纸底 + 印章/邮戳细节 + 玻璃面板**，让“连接服务器 / 安装插件 / 发送消息”像“投递邮差流程”一样明确。

关键元素：

- 背景：暖纸渐变 + 低对比网格 + 轻噪点（已在 `src/App.vue` 有类似实现）。
- 强调色：青绿（品牌/可操作）+ 焦橙（警示/强调）双轴。
- 形状：大圆角 + 胶囊按钮 + “邮戳”小标记（如 required、官方频道）。
- 字体：标题“更有情绪”，正文“更可读”；中文优先。

## 3. 组件语气（UI 文字与状态）

- **动作优先**：错误提示必须给下一步（重试/编辑/去安装）。  
- **原因可读**：连接失败/登录失败区分原因（PRD 7.3）。  
- **弱打扰提示**：断线重连使用弱提示条，不打断输入（PRD 7.3）。
