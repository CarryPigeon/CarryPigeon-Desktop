## 3. 各 Feature 设计介绍（当前实现）

### 3.1 account（身份域）
定位：统一承载认证、当前用户、用户资料能力，替代旧 `auth + user` 拆分。

结构：
1. `auth-flow`：验证码登录、required gate、鉴权错误模型。
2. `current-user`：当前登录用户展示态。
3. `profile`：资料读写用例与 port。

对外入口：`src/features/account/api.ts`
边界：跨 feature 通过 `account/api.ts` 获取能力，不直接引用内部子目录实现。

### 3.2 chat（聊天域）
定位：频道/消息/读状态/治理能力的主业务域，也是插件 domain renderer/composer 的主要宿主。

结构：
1. `room-session`：会话上下文与连接阶段编排。
2. `message-flow`：消息流、发送、渲染扩展点。
3. `room-governance`：成员/申请/禁言/治理。
4. `presentation`：MainPage、组件、store、composables。

对外入口：`src/features/chat/api.ts`
边界：跨 feature 只走 `chat/api.ts`；`domain/ports`、`data`、`presentation/store` 默认内部实现。

### 3.3 plugins（插件域）
定位：插件目录聚合、生命周期命令、运行时加载、domain registry。

结构：
1. `domain`：plugin catalog/install/runtime 的 ports 与用例。
2. `data`：HTTP/Tauri 适配。
3. `presentation/store`：catalog/install/domain registry 状态。
4. `presentation/runtime`：runtime gateway + host API + module normalize + 编排。

对外入口：`src/features/plugins/api.ts`
边界：跨 feature 仅通过 `plugins/api.ts`，API 层对外暴露只读状态 + 命令方法。

### 3.4 server-connection（连接域）
定位：服务器选择、连接状态、连接动作、server-info 的统一连接域。

结构：
1. `rack`：服务器列表、socket、TLS 策略、管理页。
2. `context`：当前服务器上下文组合层。
3. `connectivity`：连接阶段、失败原因、重试与 runtime。
4. `server-info`：服务端信息拉取缓存。

对外入口：`src/features/server-connection/api.ts`
边界：聚合导出，替代旧 `network + servers` 分裂入口。

### 3.5 settings（设置域）
定位：配置读取/写入用例与设置页 UI；当前包含 server scope 清理编排能力。

结构：
1. `domain`：ConfigPort + 配置用例（例如主题）。
2. `data`：local storage 等配置持久化。
3. `presentation`：设置页面模型与页面。
4. `application`：跨模块清理编排（`ClearServerScopeUseCase`）。

边界：当前无稳定 `api.ts`，默认不作为跨 feature 公开入口。
